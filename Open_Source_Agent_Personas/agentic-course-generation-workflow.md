# Agentic Course Generation Workflow

**[VERSION: 6.0.0]**

**Triggered via:** `/run agentic-course-generation <file reference OR pasted outline>`

This workflow forces the AI agency to execute the [Your Organization] course generation through a strict, deterministic assembly line. The AI MUST follow these steps sequentially and is explicitly forbidden from generating all 12 modules at once.

If no outline is provided (either as an attached file or pasted directly into the prompt), immediately stop and ask the user to provide the course outline before proceeding.

## PHASE 1: Scaffolding and Strategy

1. **Load the Strategic Compact & Core Doctrine:**
   - Immediately use your file reading tools to access and read `../../Strategic_Compact.md` (located in the root project folder).
   - This document contains the official Mission, Vision, and Values, the Cross-Sport logic, and the Biological System Evaluation rules. You MUST adhere to these frameworks throughout the entire generation process, prioritizing holistic safety over basic compliance.

2. **Source Ingestion (Read the Outline & Auxiliary Documents):**
   - Read the provided outline completely (whether from an attached file or pasted text). 
   - Verify it contains content suitable for exactly 12 modules.
   - Wait for and enthusiastically accept any auxiliary baseline materials the developer provides (e.g., existing syllabi, rubrics). These materials supersede generic web-research.

3. **The Deep Research Sweep (Researcher):**
   - The Researcher agent MUST execute a native web-search specifically looking for the most current Open Access sports medicine abstracts and regulatory updates related to the outline.
   - The Researcher compiles a "Dynamic Trust List" of 3-4 reputable, official organizational domains (e.g., CDC) and performs iterative deep sweeps against those specific sites.
   - **[HUMAN IN THE LOOP - DISCOVERY PAUSE]**: Stop execution. Show the developer the domains found and ask: *"Do you approve adding these organizational domains to our Deep Research Trust List?"*
   - Once approved, compile the findings into a 1-page Research Dossier.

4. **Phase 1 Vision Calibration (GraphicDesigner):**
   - The GraphicDesigner MUST verify if `Style_Book.md` exists in the local directory.
   - If missing, stop execution. Ask the developer: *"Please drop 1-3 example images representing your exact visual aesthetic into the chat."*
   - Once provided, use your Vision capabilities to extract the aesthetic DNA and mathematically generate `Style_Book.md`.

5. **Blueprint Strategy (CourseDesigner & Researcher):**
   - Utilizing the Research Dossier, the Course Designer creates a brief dual-delivery (Live/Async) strategy.
   - The AI drafts the `Master Facilitator Guide` and `Master Participant Guide` foundations.

6. **[HUMAN IN THE LOOP - PAUSE]**
   - **Stop execution.** 
   - Show the user the strategy and guide foundations. 
   - Ask: *"Does the educational outline and module breakdown look correct? Additionally, do you have specific quiz questions, answer keys, grading rubrics, or any external baseline sourcing materials (such as academic course material) you would like me to process before we begin? Please provide them now so I can anchor the course to them. Once you are ready, type 'Approve' to begin the Module Generation Assembly Line."*

## PHASE 2: The Module Assembly Line

*The AI must execute Steps 6 through 11 repeatedly for Module 1, then Module 2, all the way to Module 12. Do not proceed to the next module until the current module is approved.*

6. **Construct Content (TechnicalWriter):**
   - Generate the specific HTML lesson file for this module using the strict Freshman-College formatting rules and Mandatory Inline CSS.
   - **[MID-GENERATION CHUNKING]:** If the HTML for this lesson is massive (e.g. spanning more than 3 distinct topics/pages), you MUST NOT print all the HTML in one single block. Print the first half of the HTML, then STOP and say: *"Halfway through Lesson HTML. Type 'Continue' to generate the rest."* This prevents the output token limit from crashing mid-code.
   - Generate any required Standalone PDF Handout HTML files.
   - **[FLUSH PAUSE]:** Stop execution immediately after generating the HTML/Handouts. Do NOT generate the Assessments yet. Explicitly ask the human to type "Continue" to flush the output token budget before proceeding.

7. **Native Image Synthesis (GraphicDesigner):**
   - The GraphicDesigner reviews the HTML generated by the Technical Writer and identifies all required `[UPLOAD_URL_HERE: filename.png]` placeholders.
   - For every placeholder, the GraphicDesigner must use its native `generate_image` tool to physically render the PNG/SVG, strictly conforming to the aesthetic rules in `Style_Book.md`.
   - Save these generated images to the local `images/` director for the module.

8. **Construct Assessments & H5P (AssessmentExpert):**
   - Generate the Moodle XML quiz for this module (Strictly Matching, T/F, or Essay. NO Multiple Choice).
   - Generate the H5P scripting.

9. **Automated Internal Verification Loop (QA_Agent):**
   - **DO NOT SHOW THE CODE TO THE HUMAN YET.**
   - The QA_Agent must internally scan the generated HTML and XML against its 12-point protocol, explicitly focusing on Image Parity, external link bans, and multiple-choice bans.
   - **[SELF-HEALING OVERRIDE]**: If the QA Agent finds a formatting or tag error, it MUST NOT demand a full rewrite from the Technical Writer. It must execute a surgical, localized text replacement to fix the error in the file directly.
   - It must log its surgical corrections utilizing the `[QA AUDIT LOG]` format.

10. **Preliminary File Save:**
   - Use your file writing tools to definitively save Module [N]'s files (HTML, XML, Handouts) into the directory: `Course_Output_[CourseName]/Module_[N]/`

11. **The Diagnostic Report & Telemetry Update [HUMAN IN THE LOOP - PAUSE]:**
   - **TELEMETRY LOGGING:** Before printing the report, use your file writing tools to append any `[QA AUDIT LOG]` correction entries to `Course_Output_[CourseName]/Course_Telemetry_Log.md`.
   - Print a "Diagnostic Report" for the human referencing the saved files and ALL Audit Logs:
     > **Module [N] Diagnostic Report:**
     > ✅ Automated Verification Loop completed. 
     > [List any QA AUDIT LOG printouts of surgically repaired tokens here]
     > ✅ FilterCodes Confirmed: `{{ }}` tags and Moodle metadata correctly embedded.
     > ✅ Image Parity confirmed.
     > 📂 **Review the Files Here:** `Course_Output_[CourseName]/Module_[N]/`
   - **Stop execution.** Ask the user: *"Please visually verify the files saved in the directory above. If flawless, type 'Approve' to lock them in and advance. If you spot an error, provide specific feedback here."*
   - **ADAPTIVE LEARNING OVERRIDE:** If the human replies with corrective feedback instead of "Approve", the AI MUST:
     1. Analyze the feedback and completely rewrite the broken file for approval.
     2. Immediately append the exact Human-in-the-Loop feedback text to `Course_Output_[CourseName]/Course_Telemetry_Log.md` to permanently record the developer's correction.

12. **Advance:**
   - Upon final human approval, increment the loop to Module [N+1].

## PHASE 3: Introspective Post-Mortem (Self-Evolution)

*This phase MUST trigger immediately after Module 12 is finalized and approved.*

1. **The Telemetry Analysis:**
   - Open and read the entire `Course_Telemetry_Log.md` constructed over the 12 modules.
   - Mathematically analyze both the internal QA Agent errors AND the Human-in-the-Loop developer feedback.
   - Identify systemic failures. (e.g., *Did the Technical Writer drop Alt-Tags 4 times? Did the Course Designer keep using the wrong tone requiring 5 developer corrections? Did Developer feedback occasionally conflict with `SKILL.md` rules?*)

2. **The Evolution Proposal [HUMAN IN THE LOOP - PAUSE]:**
   - Present a structured debrief to the developer detailing the systemic failures.
   - Propose specific, permanent text modifications to the underlying Persona files (e.g., `TechnicalWriter.md`) or the master `SKILL.md` constitution to prevent these errors in future courses. 
   - Ask the developer: *"Do you approve these specific rule updates to my permanent instructions?"*

3. **Self-Modification Execution:**
   - Upon developer approval, use your file-editing tools to aggressively overwrite the local persona directories with the new, mathematically optimized rules. 
   - Congratulate the user on successfully packaging and simultaneously evolving the [Your Organization] Course Agency.

**End of Workflow.** When all 12 modules are finalized, congratulate the user.
